# MCP 网关 Nacos 服务发现需求

## 背景

明源云 AI 升级项目中，需要基于 MCP 协议建设统一工具层，将知识库、审批、文档等能力封装为 MCP Tool。为了让工具服务可扩展、可治理、可动态上下线，需要设计 MCP Server 自动注册到 Nacos，并由 MCP 网关统一发现、聚合和调度。

## 目标

- MCP Server 启动后自动注册到 Nacos，注册信息包含服务实例、协议端点、工具元数据和健康状态。
- MCP 网关从 Nacos 动态发现 MCP Server，形成统一 Tool Catalog，对上游 AI 应用提供工具发现和调用入口。
- 网关支持按工具名称、能力标签、版本、租户和健康状态选择后端 MCP Server。
- 支持 MCP Server 上下线、灰度发布、实例异常剔除和元数据变更刷新。
- 为知识库、审批、文档等 Tool 提供统一鉴权、审计、限流和调用观测能力。

## 范围

- MCP Server 注册模型与 Nacos 元数据约定。
- MCP 网关发现、缓存、工具目录聚合和调度流程。
- 工具调用链路、健康检查、故障降级、鉴权审计和可观测性设计。
- 与上游 AI 应用和下游 MCP Server 的边界契约。

## 非目标

- 不在本阶段实现具体知识库、审批、文档 Tool 的业务逻辑。
- 不替代 Nacos 的注册中心能力，不自研独立注册中心。
- 不在本阶段确定具体编程语言、框架版本和部署脚本。
- 不覆盖非 MCP 协议工具的完整迁移方案，仅预留适配能力。

## 关键角色

- 上游 AI 应用：需要发现和调用工具的业务应用或 Agent 平台。
- MCP 网关：统一接入层，负责工具发现、聚合、路由、治理和审计。
- MCP Server：承载一个或多个 MCP Tool 的后端工具服务。
- Nacos：服务注册发现与元数据配置中心。

## 验收标准

- 任一 MCP Server 启动后可自动向 Nacos 注册实例和工具元数据。
- MCP 网关无需重启即可发现新增、下线或变更的 MCP Server。
- 上游可通过网关查询统一工具列表，并按 MCP 协议调用目标工具。
- 当某个 MCP Server 实例不健康时，网关不再向该实例转发新调用。
- 调用链路具备基础日志、指标、Trace、审计记录和错误码映射。
